# Atlas API

## Objetivo

Este documento descreve como todos os componentes do Atlas se comunicam entre si.

O objetivo é definir o fluxo de informações entre WhatsApp, n8n, Atlas Core, OpenAI e Supabase.

---

## Componentes

- WhatsApp
- Evolution API
- n8n
- Atlas Core
- OpenAI
- Supabase

---

↓

↓

↓

↓

Resposta ao cliente

---

## Entrada da Mensagem

Quando um cliente envia uma mensagem pelo WhatsApp:

1. A Evolution API recebe a mensagem.
2. A Evolution envia um Webhook para o n8n.
3. O n8n inicia o workflow principal.
4. O Atlas Core recebe os dados da conversa.

---

## Dados recebidos

O Atlas deve receber pelo menos:

- company_id
- customer_name
- customer_phone
- message
- timestamp

---


## Processamento

Após receber a mensagem, o Atlas executa a seguinte sequência:

1. Identificar a empresa.
2. Buscar configurações da empresa.
3. Buscar serviços cadastrados.
4. Buscar informações na base de conhecimento.
5. Construir o Prompt.
6. Salvar a conversa.
7. Salvar a mensagem recebida.
8. Consultar a OpenAI.
9. Salvar a mensagem gerada.
10. Responder o cliente.

---

## Regras de decisão

O Atlas sempre deve seguir esta ordem antes de responder um cliente.

### 1. Identificar a empresa

Localizar a empresa responsável pela conversa utilizando o company_id.

### 2. Buscar informações

Consultar:

- Configurações da empresa
- Serviços cadastrados
- Base de conhecimento

### 3. Avaliar a pergunta

Determinar se a resposta pode ser encontrada utilizando apenas a base de conhecimento.

### 4. Consultar a OpenAI

Caso necessário, enviar contexto para a OpenAI gerar uma resposta natural.

### 5. Registrar a interação

Salvar a mensagem recebida e a resposta gerada no banco de dados.

### 6. Responder o cliente

Enviar a resposta pelo WhatsApp.
